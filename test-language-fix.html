配置页面语言切换修复测试

修复内容总结

  • 移除重复的语言设置调用：从 theme-manager.ts 中移除了语言设置，避免与手动切换冲突
  • 优化响应式语句：合并了语言初始化和主题应用逻辑，避免竞态条件
  • 增强错误处理：为所有关键函数添加了 try-catch 错误处理
  • 添加防护措施：为模型配置和语言设置添加了空值检查和后备文本
  • 改进状态管理：确保 languageInitialized 状态正确管理

主要问题分析

问题1：语言切换冲突
原因：多个地方同时调用 setLanguage()，造成状态冲突
修复：移除 applyTheme() 中的语言设置，只在用户手动切换时调用
问题2：响应式语句竞态
原因：多个响应式语句可能同时触发，导致状态不一致
修复：合并相关逻辑，添加状态检查
问题3：模型配置消失
原因：当语言切换出现问题时，相关变量变成 undefined
修复：添加空值检查和后备文本

测试步骤

  1. 重新加载扩展（在 chrome://extensions/ 页面点击刷新按钮）
  2. 打开配置页面
  3. 尝试切换中英文语言
  4. 检查是否出现以下问题：
    • 语言切换按钮无响应
    • 切换后其他配置项失效
    • 模型配置消失
    • 默认模型选择器消失
  5. 多次快速切换语言，测试稳定性

预期结果

✅ 语言切换应该立即生效
点击语言按钮后，界面文字应该立即切换到对应语言
✅ 其他配置项保持正常
语言切换后，模型配置、默认模型选择等功能应该继续正常工作
✅ 多次切换稳定
连续多次切换语言不应该导致页面功能失效
✅ 控制台无错误
打开开发者工具，应该看到正常的日志信息，没有错误

关键修改文件

  • src/options/OptionsApp.svelte - 优化响应式逻辑和错误处理
  • src/lib/i18n/index.ts - 增强语言切换的错误处理
