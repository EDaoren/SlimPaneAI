🔍 问题分析

设置页面的左侧导航按钮国际化不生效，主要原因是：

  • 设置页面没有初始化国际化系统
  • 用户偏好设置加载后，没有调用语言初始化函数
  • 语言切换时没有立即更新界面
🔧 解决方案

通过以下修改解决问题：

  • 导入 initializeLanguage 和 setLanguage 函数
  • 在 onMount 中初始化语言设置
  • 在语言切换时立即更新界面

🔧 具体修复内容

// 1. 导入国际化函数 import { t, initializeLanguage, setLanguage } from '@/lib/i18n'; // 2. 在 onMount 中初始化语言 onMount(async () => { await settingsStore.loadSettings(); const currentSettings = settingsStore.getCurrentState(); if (currentSettings.userPreferences) { applyTheme(currentSettings.userPreferences); // ✅ 初始化国际化系统 initializeLanguage(currentSettings.userPreferences); } }); // 3. 语言切换时立即更新界面 async function handleLanguageChange(language: 'en' | 'zh') { const newPreferences = { ...userPreferences, language }; await settingsStore.saveUserPreferences(newPreferences); // ✅ 立即切换界面语言 setLanguage(language); }

✨ 修复成果

  • 正确初始化 - 设置页面加载时根据用户偏好设置语言
  • 实时切换 - 语言变化立即反映在导航按钮上
  • 完整同步 - 设置页面和侧边栏语言保持一致
  • 持久化存储 - 语言偏好正确保存和加载
  • 用户体验 - 无需刷新页面即可看到语言变化

🎉 修复完成

现在设置页面的左侧导航按钮能够正确响应语言设置， 用户选择英文时会立即显示英文文本，选择中文时显示中文文本。 这确保了完整的国际化体验，让全球用户都能享受本地化的界面。
