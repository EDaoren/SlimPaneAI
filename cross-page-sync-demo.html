通过 Chrome Storage API 实现跨页面实时同步

🔧 技术实现方案

⚙️

→
💾

📡

🎨

应用新主题
// 1. 侧边栏添加存储监听器 chrome.storage.onChanged.addListener(handleStorageChange); // 2. 处理存储变化 function handleStorageChange(changes) { if (changes.userPreferences) { const newPreferences = changes.userPreferences.newValue; // 强制刷新设置并应用主题 settingsStore.forceRefresh(); applyTheme(newPreferences); } } // 3. 响应式主题应用 $: if ($settingsStore.userPreferences) { applyTheme($settingsStore.userPreferences); }

✨ 修复成果

  • 实时同步 - 设置页面的主题变化立即同步到侧边栏
  • 双向响应 - 侧边栏设置也能同步到设置页面
  • 无需刷新 - 所有变化都是实时的，无需手动刷新
  • 一致体验 - 确保所有页面的主题保持一致
  • 可靠机制 - 基于 Chrome Storage API 的稳定同步
  • 性能优化 - 只在设置真正变化时才触发更新

🎯 用户体验提升

现在用户在任何页面更改主题设置时，所有相关页面都会立即同步更新。 无论是在设置页面切换主题，还是在侧边栏调整偏好， 整个扩展的界面都会保持一致的视觉风格，提供无缝的用户体验。
